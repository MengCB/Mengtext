 代码规范专题
	 1.顶格缩进
	   顶格: 定义变量, 定义函数, 执行单条语句,单行注释
	   缩进: 函数体内容相对函数声明,循环体,分支程序,
	 2.空格 运算符左右, 注释前,注释后, 逗号后(分隔符后)
	 3.分隔符号紧跟代码, : ,后跟一个空格
           逗号, 分号, 紧跟代码
           左括号紧跟,调用函数的情况
         4.换行 多条语句（遇到分号),否则很难进行代码调试  注释语句后必须换行 } 例外
	 5.注释的规范 声明函数前,应该注释   多个逻辑分支并列,应该注释
	   重要的核心算法,复杂的算法 要注释
	   逻辑变更,修改记录,应该注释
	 6.命名规范
	   见名知意 类名对象构造器函数首字母大写
	   驼峰命名法(国际)  loadLocalSetings // load_local_settings 用于常量//
	   严禁滥用汉语拼音!!!
	   尽量不要用数字和特殊符号
	   全局变量-必须有标记 _doRain
	   其他约定-temp临时变量 flag标志位 counter timmer计时器或定时器 parameter参数
	 
       二: debug 语法错误, 逻辑错误!!
	 1. 定位问题 (定位问题 = 解决问题)  看报错信息或输出是否正确
	 2. 不确定 = 不可靠 (什么是确定？ 以程序为准，不是主观判断)
	 3. 无语法错误的时候,依然要检索逻辑错误
